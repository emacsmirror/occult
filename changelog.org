#+TITLE: occult.el changelog
#+AUTHOR: Ag Ibragimov

Changes to the project, following Keep a Changelog conventions.

* [1.3.0] - 2026-04-20

** Changed
- ~occult-hide-region~ now absorbs any existing folds overlapping the region instead of signaling ~user-error~. Bounds of touched folds extend the new fold so no hidden content is lost. Selecting a region that wraps or crosses an existing fold now collapses the combined span into a single fold.

* [1.2.0] - 2026-04-11

** Fixed
- Folding a region that begins with blank lines no longer collapses the summary to just the ellipsis. A new head overlay hides the leading whitespace so the summary shows the first non-whitespace line.

** Changed
- ~occult-summary-max-length~ is now measured from the first non-whitespace position of the region rather than the region start. Leading blank lines no longer consume any of the summary budget.
- Fold overlay structure is now three overlays: parent, head, and body. The indicator ~before-string~ lives on the head overlay so its placement is uniform regardless of leading whitespace. Parent and head are non-evaporating so a region edit cannot drop them before the modification-hook cleans up.

Investigated and initially prototyped by @ImFstAsFckBoi in PR #2.

* [1.1.0] - 2026-04-05

** Added
- ~occult-edit-region~ command - edit or view the content of a fold in a narrowed indirect buffer. Bound to ~e~ on the fold keymap.
- ~occult-edit-mode~ minor mode with a dynamic header line that always reflects the current commit and abort key bindings.
- ~occult-edit-commit~ and ~occult-edit-abort~ commands. Commit keeps user changes; abort restores the original fold content via ~replace-buffer-contents~ without dissolving the fold overlays.
- View-only session for read-only base buffers. Header shows "View Occult Fold" with a single close key; both commands simply close the view buffer.
- Customizable key bindings: ~occult-edit-commit-key~ (default ~C-c C-c~) and ~occult-edit-abort-key~ (default ~C-c C-k~).
- New faces for the edit header line: ~occult-edit-header~, ~occult-edit-commit-key~, ~occult-edit-abort-key~, ~occult-edit-header-separator~.

* [1.0.0] - 2026-04-05

** Added
- Region is restored after expanding an individual fold via ~occult-toggle~. The mark and point snap back to the fold's original boundaries on expand. ~occult-reveal-all~ is unaffected.

* [0.1.0] - 2026-03-25

** Added
- ~occult-toggle~ command - collapse region or expand fold at point
- ~occult-reveal-all~ command - expand all folds in buffer
- ~occult-hide-region~ programmatic API
- Two-overlay fold structure: first line stays visible and navigable
- Isearch integration with temporary and permanent reveal
- Evil ex-search integration (optional, loaded lazily)
- ~revert-buffer~ persistence via content hash verification
- ~occult-auto-reveal~ option (nil, echo, expand)
- Customizable indicator, ellipsis, and summary length
- Works in read-only and special-mode buffers
